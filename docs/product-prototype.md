# 首版产品原型说明

本文档用于描述“网站分身管理器”V1 的界面结构、交互路径和页面组成，方便继续迭代 UI、补设计稿或对接开发。

## 产品定位

网站分身管理器不是一个通用浏览器，而是一个“网站多账号登录工作台”。

它的核心价值是：

- 让一个站点下的多个账号分身互相隔离
- 让用户用更低成本切换不同网页登录身份
- 让站点管理、分身管理和实际网页登录使用放在一个桌面工具里完成

## 主工作台结构

V1 支持三种布局模式，它们共用同一套数据结构和运行时逻辑。

### 1. 工作台模式 `workspace`

适合日常重度使用，是默认模式。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部栏：应用标题 / 站点与分身入口 / 设置入口                                 │
├───────────────┬───────────────────────┬──────────────────────────────────────┤
│ 站点管理区     │ 分身管理区             │ 浏览器工作区                         │
│               │                       │                                      │
│ - 站点树       │ - 分身列表             │ - 布局切换                           │
│ - 新建站点     │ - 新建分身             │ - 默认打开模式切换                   │
│ - 编辑/删除站点│ - 分身状态标签         │ - 内嵌会话标签                       │
│ - 站点备注信息 │ - 内嵌打开/兼容打开    │ - 地址栏 / 前进 / 后退 / 刷新        │
│               │ - 编辑/删除分身        │ - 外置打开 / 清空数据 / 关闭分身     │
│               │                       │ - 内嵌 webview 浏览区域              │
└───────────────┴───────────────────────┴──────────────────────────────────────┘
```

### 2. 分屏页模式 `split-page`

适合希望“先管理、后使用”的场景。

```text
管理页：
┌────────────────────────────────────────────────────────────┐
│ 顶部栏                                                     │
├───────────────────────┬────────────────────────────────────┤
│ 站点管理区             │ 分身管理区                        │
└───────────────────────┴────────────────────────────────────┘

浏览页：
┌────────────────────────────────────────────────────────────┐
│ 返回管理页 / 当前分身信息                                   │
├────────────────────────────────────────────────────────────┤
│ 浏览器工具栏 + 内嵌浏览区域                                 │
└────────────────────────────────────────────────────────────┘
```

### 3. 标签页模式 `tabbed`

适合把产品更偏向“浏览器工作台”的用户。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 顶部栏：管理站点与分身 / 会话数量 / 当前标签                                  │
├──────────────────────────────────────────────────────────────────────────────┤
│ 浏览器区域：                                                                   │
│ - 顶部会话标签                                                                  │
│ - 地址栏和浏览操作                                                              │
│ - 内嵌浏览内容                                                                  │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 主要页面与模块

### 站点管理模块

负责维护“要管理哪些网站”。

字段：

- 站点名称
- 站点类型
- 首页 / 登录入口 URL
- 站点目标规则
- 备注

站点类型：

- `domain`
  - 按域名管理
- `entry_url`
  - 按完整登录入口管理
- `group`
  - 多域名 / 多入口归为一个站点组

站点操作：

- 新建站点
- 编辑站点
- 删除站点
- 查看站点备注与入口地址

### 分身管理模块

负责维护“同一个站点下有哪些账号环境”。

字段：

- 分身名称
- 所属站点
- 备注

状态：

- `未打开`
- `内嵌中`
- `外置中`

分身操作：

- 新建分身
- 编辑分身
- 删除分身
- 内嵌打开
- 兼容模式打开

### 浏览器工作台模块

负责实际网页登录和切换使用。

工具栏能力：

- 地址栏访问
- 前进
- 后退
- 刷新
- 回到首页
- 外置打开
- 清空分身数据
- 关闭分身

会话区能力：

- 显示当前所有内嵌会话
- 切换当前激活分身
- 显示当前所有外置兼容会话

浏览区能力：

- 渲染内嵌 webview
- 随布局变化自动调整区域尺寸
- 同一时间仅显示一个激活内嵌分身

## 关键交互流程

### 1. 创建站点并创建分身

```text
新建站点
  -> 输入站点名称与入口地址
  -> 选择站点类型
  -> 保存
  -> 在该站点下新建分身
  -> 输入分身名称与备注
  -> 保存
```

### 2. 打开分身登录

```text
选择站点
  -> 选择分身
  -> 点击“内嵌打开”或“兼容打开”
  -> 创建独立 webview / webview window
  -> 使用该分身完成网页登录
  -> Cookie 与本地存储落到独立目录
```

### 3. 重复打开已存在分身

```text
若同一分身已打开：
  - 内嵌分身：直接切换到对应会话
  - 外置分身：直接聚焦已有窗口
  - 不重新创建新实例
```

### 4. 清空分身状态

```text
选择分身
  -> 点击“清空数据”
  -> 关闭当前分身实例
  -> 删除该分身的本地浏览数据目录
  -> 下次打开时以全新状态进入
```

## 界面风格说明

V1 当前实现采用轻桌面工作台风格：

- 顶部深色渐变头部
- 主体使用浅色玻璃卡片风格
- 强调“管理面板 + 工作区”的布局层次
- 使用 Ant Design 组件维持开发效率

后续可以继续增强：

- 更明确的站点图标系统
- 最近使用排序
- 分身颜色标识
- 更强的会话标签视觉区分
- 站点分组与搜索
